1 Get API Key

Before you begin, you need to obtain an API key. Please contact the administrator to get your dedicated API key.

API key format: sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

Please keep your API key secure. Do not share it with others or commit it to public code repositories.

2 API Endpoints

We provide two API formats. You can choose according to your needs:

Claude API Format

POST {BASE_URL}/v1/messages

OpenAI API Format

POST {BASE_URL}/v1/chat/completions
Replace {BASE_URL} with the actual service address, e.g., http://localhost:3000 or your deployment domain.

1 What is CCswitch

CCswitch is a Claude Code / Codex proxy switching tool that provides a visual interface to manage multiple API provider configurations, supporting one-click switching between different API endpoints.

Key Features:

  • Support for Claude Code, Codex and other clients
  • Visual configuration management interface
  • Built-in provider templates (DeepSeek, GLM, ModelScope, etc.)
  • Custom provider configuration support
  • Configuration import/export for easy backup and migration
CCswitch project: https://github.com/yiGmMk/ccswitch

2 Install CCswitch 

go install github.com/yiGmMk/ccswitch@latest

Download the precompiled binary for your platform from GitHub Releases:

# Visit the download page
https://github.com/yiGmMk/ccswitch/releases

3 Configuration File Location

CCswitch configuration files are stored in the following locations:

Client Config Directory
CCswitch ~/.ccswitch/
Claude Code ~/.claude/
Codex ~/.codex/
CCswitch supports configuration import/export for easy backup and migration to other devices.

4 Add Provider Configuration

After launching CCswitch, click the [Add Provider] button. You can choose between two configuration methods:

CCswitch has built-in provider templates. Selecting a template will auto-fill the configuration:

  • DeepSeek
  • GLM (Zhipu)
  • ModelScope
  • More templates coming soon...

After selecting a template, just enter your API Key to complete the configuration.

If there is no corresponding provider template, you can select [Custom Configuration]:

# Custom provider configuration example
Name: Kiro API
Base URL: {BASE_URL}
API Key: sk-your-api-key

Replace {BASE_URL} with the actual service address.

Click [Add] to save after configuration. The configuration will appear in the home page list.

5 Enable Configuration

Click the [Enable] button in the configuration list to activate the corresponding provider:

  1. Find the provider configuration you added on the home page
  2. Click the [Enable] button on the right side of the configuration item
  3. After successful activation, restart Claude Code as prompted
  4. After restart, you can use the new API endpoint
You can add multiple provider configurations and switch between different API services at any time.

1 Install Claude Code

Claude Code is Anthropic's official command-line tool, which can be installed globally via npm:

npm install -g @anthropic-ai/claude-code
Please ensure Node.js 18 or higher is installed before installation. Check your version with node -v.

2 Configure Environment Variables

Set the API key and service endpoint to connect Claude Code to this service:

Add to ~/.bashrc or ~/.zshrc

export ANTHROPIC_API_KEY="sk-your-api-key"
export ANTHROPIC_BASE_URL="{BASE_URL}"

After saving, run source ~/.bashrc or source ~/.zshrc to apply the configuration.

PowerShell - Add to $PROFILE

$env:ANTHROPIC_API_KEY = "sk-your-api-key"
$env:ANTHROPIC_BASE_URL = "{BASE_URL}"

Or set via system environment variables (permanent):

[System.Environment]::SetEnvironmentVariable("ANTHROPIC_API_KEY", "sk-your-api-key", "User")
[System.Environment]::SetEnvironmentVariable("ANTHROPIC_BASE_URL", "{BASE_URL}", "User")
Replace {BASE_URL} with the actual service address, and sk-your-api-key with your API key.

3 Start Your First Session

Open a terminal in any project directory and start Claude Code:

cd /path/to/your/project
claude

You will see the Claude Code welcome screen with your session info, recent conversations, and latest updates.

Type /help to view available commands, or /resume to continue a previous conversation.

4 Basic Usage

Claude Code supports multiple usage methods:

# Start interactive session
claude

# Send message directly
claude "Hello, please introduce yourself"

# Use pipe input
cat file.txt | claude "Please analyze this file"

# Resume previous conversation
claude --resume

5 Common Commands

Command Description
claude Start interactive session
claude "message" Send a single message and get a response
/help View all available commands
/resume Resume previous conversation
/clear Clear current session
/login Re-login or switch account
claude --version View version information

1 What is opencode

opencode is an open-source AI programming assistant that supports multiple LLM backends. With configuration, you can use the Claude models provided by this service.

opencode project: https://github.com/opencode-ai/opencode

2 Install opencode

Install opencode using a package manager:

brew install opencode-ai/tap/opencode
# Download latest version
curl -fsSL https://github.com/opcode-ai/opencode/releases/latest/download/opencode_linux_amd64.tar.gz | tar xz
sudo mv opencode /usr/local/bin/
go install github.com/opencode-ai/opencode@latest

3 Configure opencode

Create or edit the configuration file ~/.config/opencode/config.toml:

[providers.anthropic]
api_key = "sk-your-api-key"
base_url = "{BASE_URL}"

[agents.coder]
model = "anthropic:claude-sonnet-4-20250514"
max_tokens = 8192

[agents.task]
model = "anthropic:claude-sonnet-4-20250514"
max_tokens = 8192
Replace {BASE_URL} with the actual service address, and sk-your-api-key with your API key.

4 Use opencode

Start opencode in your project directory:

# Enter project directory
cd your-project

# Start opencode
opencode

opencode will automatically read the configuration file and connect to this service.
